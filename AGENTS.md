# Repository Guidelines

此仓库按章节整理 Bash 与 shell 脚本示例和笔记。贡献时请保持按章节组织内容，并以可独立运行的小脚本为主。

## 项目结构与模块组织

- 章节内容位于 `chNN-*` 目录（如 `ch12-*`），每章都有自己的 `README.md`。
- 脚本（`*.sh`）、数据文件（`*.txt`、`*.csv`）与图片（`*.png`）放在对应章节目录内。
- 根目录 `README.md` 汇总各章节链接；新增或重命名章节内容时要同步更新。

## 构建、测试与开发命令

- 本仓库没有统一的构建系统，脚本均可独立运行。
- 运行脚本：`bash path/to/script.sh`（路径含空格时请加引号）。
- 需要时可赋予可执行权限：`chmod +x path/to/script.sh`。
- 快速搜索内容：`rg "pattern" chNN-*`。

## 编码风格与命名约定

- 脚本头部使用 Bash shebang（`#!/bin/bash`）。
- 缩进优先使用空格，并保持与当前文件一致。
- 脚本命名简短且描述性强（如 `test1.sh`、`check_file_dates.sh`）。
- 无明确理由不要重命名章节目录或跨章节移动文件。

## 测试指南

- 本仓库没有自动化测试框架。
- 通过运行脚本并对比示例输出文件（如 `*.out`、`data*`）验证变更。
- 新增脚本时，请在脚本附近提供最小化的示例输入。

## 提交与 PR 指南

- 提交信息简短、动作导向，常包含章节名（如 "update ch17"、"add directory"）。
- PR 需包含简要说明、涉及的章节，以及验证方式（运行的命令、OS/shell 版本）。
- 仅在脚本产生界面输出时添加截图（如 GUI 章节的对话框示例）。

## 安全与环境说明

- 部分脚本使用绝对路径（如 `/archive/*`）；所需目录或权限请写明在对应章节的 `README.md`。
- 避免在真实系统上直接运行具有破坏性的脚本，优先使用测试目录。

## 贡献者注意事项

- 除非必要，避免提交体积较大的生成二进制文件。
- 章节笔记应保持简洁，并与对应代码示例一致。
- 本仓库主要用于学习。
- 始终用中文与我进行交流。
